Проект автоматизации тестирования веб-приложения Qa-desk
Данный проект содержит автоматизированные UI-тесты для веб-приложения qa-desk.stand.praktikum-services.ru. Тесты покрывают ключевые пользовательские сценарии: регистрацию, вход в систему, выход, создание объявления авторизованным и неавторизованным пользователем.

Требования к окружению
Установленный Python версии 3.8 или выше.
Браузер Google Chrome (рекомендуется последняя стабильная версия).
ChromeDriver, соответствующий версии Chrome (или использование webdriver-manager для автоматической загрузки).

Установка Selenium
терминал
pip install selenium  

Запуск тестов 
терминал
pytest -v tests/

В файле conftest.py заданы глобальные константы:

BASE_URL — адрес тестируемого стенда.

EXISTING_USER_EMAIL и EXISTING_USER_PASSWORD — учетные данные заранее созданного пользователя для тестов входа и выхода.

Примечания
Во всех тестах используются явные ожидания (WebDriverWait) с таймаутами 3–5 секунд для обеспечения стабильности.

Для кликов по элементам, которые могут перекрываться или требовать наведения, применяется ActionChains.

Локаторы хранятся отдельно, что облегчает поддержку при изменении вёрстки.

Генерация случайных данных для регистрации гарантирует уникальность тестовых пользователей и позволяет избежать конфликтов.

Описание файлов
locators.py — содержит кортежи локаторов (By, селектор) для всех используемых элементов на страницах. Локаторы сгруппированы по разделам (главная страница, окна входа/регистрации, форма создания объявления, страница профиля).

conftest.py — глобальные фикстуры pytest:

driver — инициализация и закрытие браузера Chrome с заданным размером окна.

generate_random_email — генерация случайного email для регистрации.

generate_random_password — генерация случайного пароля.

get_credentials_email_password — создание нового пользователя и возврат его email/пароля для использования в тестах.

test_user_registration.py — тесты регистрации:

test_user_registration_successful — успешная регистрация нового пользователя.

test_registration_with_invalid_email_should_fail — попытка регистрации с некорректным email (без @) – ожидается ошибка и красная подсветка полей.

test_registration_with_existing_user_should_fail — попытка повторной регистрации уже существующего пользователя.

test_user_login.py — тест входа:

test_user_login_successful — успешный вход с заранее созданными учетными данными.

test_user_logout.py — тест выхода:

test_user_logout — выход из системы и проверка, что элементы авторизованного пользователя исчезли.

test_ad_creation_authorized.py — создание объявления авторизованным пользователем:

test_ad_creation_authorized_successful — полный сценарий: авторизация, заполнение формы, выбор категории/города/состояния, публикация, проверка появления объявления в профиле.

test_ad_creation_unauthorized.py — создание объявления неавторизованным пользователем:

test_unauthorized_ad_creation_shows_auth_modal — нажатие на кнопку «Разместить объявление» без входа приводит к появлению модального окна с заголовком.
